Программа реализует упрощённый механизм внедрения зависимостей без сторонних библиотек.

Идея: в классе можно объявить поля типов интерфейсов и пометить их аннотацией `@AutoInjectable`, при этом не создавать объекты вручную (`new`).  
Класс `Injector` сам найдёт такие поля через рефлексию, прочитает из файла `injector.properties`, какую реализацию нужно использовать, создаст объект и запишет его в поле.

Что реализовано
- Аннотация `@AutoInjectable` для пометки полей, куда нужно внедрить зависимость.
- `Injector`:
  - читает `injector.properties` через `java.util.Properties`
  - ищет в объекте поля с `@AutoInjectable`
  - определяет тип поля (интерфейс)
  - находит реализацию в properties (`interfaceFQN = implFQN`)
  - создаёт экземпляр реализации (через reflection) и записывает в `private` поле
  - печатает лог внедрения (что куда было подставлено)
- Демонстрация:
  - без инъекции вызов метода падает (поля `null`)
  - после `inject()` метод работает
  - при смене одной строки в `injector.properties` меняется поведение программы (A → B)
  - 
Минимально нужное для работы:
- `Main.java`
- `injector.properties` (создаётся автоматически при первом запуске, если отсутствует)

Файл `injector.properties` содержит пары <полное имя интерфейса>=<полное имя класса реализации>:
`Main$SomeInterface=Main$SomeImpl`
`Main$SomeOtherInterface=Main$SODoer`

Чтобы переключить поведение, достаточно заменить первую строку на эту: `Main$SomeInterface=Main$OtherImpl`
